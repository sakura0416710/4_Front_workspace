변수와 자료형

변수 선언

함수 내부에서 'var 변수명;'으로 선언하면 지역 변수가 됨
함수 내부에서 '변수명;'으로 선언하거나, 함수 밖에서 '변수명;' 혹은 'var변수명;' 으로 선언하면 전역변수 전역변수와 지역변수가 동일한 이름일 경우 함수 내부에서 변수명을 호출하면 지역변수가 우선권을 가짐.
전역변수 사용 시 'window.변수명;' 혹은 'this.변수명;'으로 표현하여 지역변수와 구분
전역변수는 해당 window내에서 어디든 사용할 수 있으며 지역변수는 해당 함수 내에서만 사용가능.

자료형

자바스크립트에서는 자료형 별로 변수 타입이 지정되지않고 리터럴에 의해 자료형 결정

typeof()연산자

값의 자료형을 확인하는 연산자 -> 선언시 자료형을 지정하지 않기 때문에 변수명만으로 데이터의 자료형을 알 수 없어 변수 자료형 확인 시 사용한다.

var와 let과 const

기존 es5 버전에서는 모든 변수를 var자료형으로 받아왔지만 데이터의 모호함이라는 문제를 개선하기 위해 최근에는 es6버전의 let과 const자료형을 많이 쓰는 추세 -> 가독성및 유지보수 증가
-let :일반적인 변수 선언 (덮어쓰기 가능, 기존 var와 같음)
-const : 상수(final)변수 선언.덮어쓰기 불가능
